枚举和注解

枚举（enumeration）

说明：

  1. 枚举是一组常量的集合
  2. 可以这样理解：枚举属于一种特殊的类，里面只包含一组有限的特定的对象

枚举的两种实现方式：

  1. 自定义类实现枚举

    1. 不需要提供 set 方法，因为枚举对象值通常为只读（构造器需要私有化）
    2. 对枚举对象/属性==使用public final + static 共同修饰==，实现底层优化 (对外暴露对象)
    3. 枚举对象名通常要使用==全部大写==，常量的命名规范
    4. 枚举对象根据需要，也可以有多个属性

  2. enum关键字实现枚举，具体规则如下：

    1. 定义的对象实质上还是public final static 类型

    2. 当用enum开发一个枚举类时，默认会继承Enum类，此时就可以==使用Enum类的方法==

    3. 如果使用==无参构造器==创建枚举对象，则==实参列表和小括号都可以省略==

    4. 使用有参构造器时，要==明确调用的是哪个构造器==

    5. 如果有多个枚举对象，使用 , 间隔，最后用一个 ; 结尾

    6. 枚举对象必须放在枚举类的==行首==

    7. enum关键字实现的枚举类==不能继承任何类==，因为在底层已经继承了Enum类，java是单继承

      但是还可以实现接口

      protected Object clone() 抛出CloneNotSupportedException。
      int compareTo(E o) 将此枚举与指定的对象进行比较以进行订购。
      boolean equals(Object other) 如果指定的对象等于此枚举常量，则返回true。
      protected void finalize() 枚举类不能有finalize方法。
      类<E> getDeclaringClass() 返回与此枚举常量的枚举类型相对应的Class对象。

      String name() 返回此枚举常量的名称，与其枚举声明中声明的完全相同。

      static <T extends Enum<T>>T valueOf(类<T> enumType, String name) 返回具有指定名称的指定枚举类型的枚举常量。

注解（Annotation）

说明：

  1. 注解（Annotation）也被称为元数据（Metadata），用于修饰解释 包、类、方法、属性、构造器、局部变量等数据信息

  2. 和注释一样，注释不影响程序执行，但注释可以被编译或运行，相当于嵌入在代码中的补充信息

  3. 在javaEE中，注解的使用目的比较简单，例如标记过时的功能，忽略警告等，在JavaEE中注解占据了更为重要的角色，例如用来配置应用程序的任何切面，代替JavaEE旧版中所遗留的繁冗代码和XML配置

  4. 使用 Annotation 时，要在前面加 @ 符号，并把 Annotation 当成一个修饰符使用，用于修饰它支持的程序元素

  5. 三个基本的 Annotation：

    1. ==@Override== 限定某个方法，是重写父类的方法，该注解只能用于方法

      如果加了此注解，编译器就会去检查该方法是否真的重写了父类的方法，如果确实重写了，则编译通过，如果没有构成重写，则编译错误 ==> 语法校验

    2. ==@Deprecated== 用于表示某个程序元素（类、方法等）已过时，即不再推荐使用

      可以做版本升级兼容过渡使用

    3. ==@SuppressWarrings== 抑制编译器警告

      当我们不希望看到警告是，可以使用这个注解来抑制警告信息

      通常我们可以放在具体的语句上，或者类上

JDK的元注解：

说明：是修饰注解的注解

四种元注解：

  1. @Retention // 指定注解的作用范围 三种（SOURCE、CLASS、RUNTIME）
  2. @Target // 指定注解可以在哪些地方使用，包含一个 values 成员变量
  3. @Documented // 指定注解是否会在 Javadoc 中体现
  4. @Inherited // 子类会继承父类的注解

异常

基本介绍：Java语言中，将程序执行中发生的不正常的情况称之为 ==异常==（语法错误和逻辑错误不算异常）

基础处理措施：==将可能出错的代码选中 —> alt + ctrl + T —> 选中 try-catch==

异常事件分类：

  1. ==Error错误==：Java虚拟机无法解决的严重问题，如：JVM系统内部错误，资源耗尽等严重情况，

    比如：StackOverflowError（栈溢出）和OOM（out of memory）==Error是严重错误，程序会崩溃==

  2. ==Exception==：其他因编译错误或者偶然的外在因素导致的一般性问题，可以使用针对性的代码进行处理，例如：空指针访问，试图读取不存在的文件，网络连接中断等等，==Exception异常分为两大类：运行时异常 [ 程序运行时发生的异常 ] 和编译时异常 [ 编程时，编译器检查出的异常 ]==
