面向对象编程（高级）

类变量和类方法

类变量/静态变量

说明：

  1. 类变量（静态变量）是所有类的示例==共享的变量==，

  2. static类变量 在==类加载==的时候就已经生成了，即使没有创建实例也能继续使用

  3. 不管存在于堆还是方法区的静态域中，都不影响使用（jdk8以前是放在方法区内，jdk8以后，是放在堆中定义类型对应的 class类 中（反射））

定义语法：

访问修饰符 static 数据类型 变量名;

访问形式：

类名 . 变量名（访问控制符和普通属性的访问修饰是相同的）

使用细节：

  1. 什么时候需要使用类变量

    当我们需要让某个类的所有对象都共享一个变量时，就可以考虑使用类变量（静态变量）：比如定义学生类，统计学生要交的学费

  2. 类变量与（示例变量）的区别

    类变量是所有对象共享的，而实例对象是每个对象独有的

  3. 加上 static 称为类变量或者静态变量，否则称为实例变量/普通变量/非静态变量

  4. 类变量可以通过 类名 . 类变量名 ，或者 对象名 . 类变量名 来访问，推荐使用前者进行访问（需要满足访问修饰符的访问权限和范围）

  5. 实例变量==不能通过 类名 . 变量名== 来访问

  6. 静态变量是类加载的时候就创建了，就算没有创建类的实例，我们也可以通过类名.类变量名来访问

  7. 类变量的生命周期是随着类的加载开始，随着类的消亡而销毁

类方法

说明：当方法被 static 修饰时，该方法就可以访问类的静态变量

​ 调用，对象名或者类名 . 方法名

使用细节：

  1. 如果我们希望不创建实例，也可以调用某种方法（即当作工具使用）

  2. 这时可以将方法设计成静态方法，提高我们的开发效率

  3. 在实际的程序开发中，往往会有一些通用的方法，设计成静态方法，这样我们就可以不创建对象就可以使用了（打印数组、冒泡排序等）即开发自己的工具类

  4. 类方法和不同方法都是随着类的加载而加载，将结构信息存储在方法区

  5. 类方法中==无 this 参数==，普通方法中==隐含着 this 参数==

  6. 类方法可以通过类名调用，也可以通过对象名调用

  7. 普通方法和对象有关，需要通过对象名调用，比如对象名.方法名（参数），不能通过类名调用

  8. 类方法中不允许使用 this 和 super 关键字

  9. ==类方法（静态方法）只能访问静态成员（静态变量或者静态方法）==

  10. 普通方法（非静态方法）可以访问所有成员

==分析代码要注意 ++ 的位置，前 ++ 或者 后 ++==

main语法说明

说明：

  1. main方法 由java虚拟机调用 ，所以该方法的访问权限必须是 public

  2. java虚拟机在执行 main方法 时，不必创建对象，所以该方法必须是 static

  3. 该方法接收 String类型 的数组参数，该数组中保存执行java命令时传递给所运行的类的参数（也就是 args 5数组中保存着运行的类所需要的参数）

  4. java 虚拟机不需要返回值，所以返回值为 void

  5. java 运行的程序 参数1 参数2 参数3

    这三个参数会导入到 args 数组 中

  6. 在 main() 方法（main方法本身就是静态方法）中，我们可以直接调用main方法所在类的静态方法或者静态成员

  7. 但不能直接访问该类中的非静态成员，必须创建一个该类的实例对象后，才能通过这个对象去访问这个类中的非静态成员

代码块（codeblock）

说明：

  1. 代码块又称为初始化块，属于类中的成员（即类的一部分），类似于方法，将逻辑语句封装在方法中，通过 { }包围起来
  2. 和方法不同，没有方法名，没有返回值，没有参数，只有方法体，而且不用通过 对象或者类 显示调用，而是==加载类时==，或创建对象时隐式调用

语法：

[修饰符]{

​ }；

注意：

  1. 修饰符 可选 ，要写的话，也只能写 static
  2. 代码块分为两类，使用 static 的叫做静态代码块，没有则为普通代码块
  3. 逻辑语句可以为任何逻辑语句（输入、输出、方法调用、循环、判断等）
  4. 代码块后 ；可以写上，也可以省略
  5. 相当于是==另外一种形式的构造器==（对构造器的补充机制），可以做初始化的操作
  6. ==使用场景==：如果多个构造器中都有重复的语句，则可以抽取到代码块中，提高代码的==复用性==

使用注意事项：

  1. static代码块也叫做静态代码块，作用就是对类进行初始化，而且它==随着类的加载而执行，并且只会执行一次==，如果是普通代码块，每创建一个对象就执行一次

  2. 类什么时候被加载

    重点：

    1. 创建对象实例时（new 一个新对象的时候）
    2. 创建子类对象实例，父类也会被加载（根据构造器原理思考，==先加载父类，在加载子类==）
    3. 使用类的静态成员时（静态属性、静态变量）

  3. 普通的代码块，在创建对象实例时，会被隐式的调用，创建一次，就会被调用一次

    如果只使用类的静态成员时，普通代码块并不会执行（因为在调用静态成员时，并不需要创建一个新的类的实例对象）

  4. 在创建一个对象时，在一个类 调用顺序是：

    1. 调用静态代码块和静态属性初始化（注意：静态代码块和静态属性初始化调用的优先级一样，如果有多个静态代码块和多个静态变量初始化，则按==他们定义的顺序调用==）
    2. 调用普通代码块和普通属性的初始化（两者的优先级相同，如果有多个，则按定义顺序调用）
    3. 调用构造方法（最后调用构造器）

  5. 构造器 的最前面其实隐含了 super() 和调用普通代码块，静态相关的代码块，属性初始化，在类加载时，就执行完毕，因此是优先于构造器和普通代码块执行的

  6. 创建一个子类时（继承关系），他们的静态代码块、静态属性初始化、普通代码块、普通属性初始化，构造方法的==调用顺序==如下：！！

    1. 父类的静态代码块和静态属性
    2. 子类的静态代码块和静态属性
    3. 父类的普通代码块和普通属性初始化
    4. 父类的构造方法
    5. 子类的普通代码块和普通属性初始化
    6. 子类的构造方法
    7. 注意：每个点的优先级一样，按照定义顺序执行

  7. 静态代码块只能直接调用静态成员（静态属性和静态方法），普通代码块可以调用任意成员

==当直接通过 类名.变量名 访问静态变量，则在运行时，会进行类的加载，则此时静态代码块和静态变量初始化都会进行==

单例设计模式

  1. 要求采取一定的方法保证在这个软件系统中，对某个类只能存在一个对象实例，并且该类只提供一个取得其对象实例的方法

  2. 单例模式有两种方式：

    1. 饿汉式
    2. 懒汉式

  3. 模式实现：

    1. 饿汉式（可能创建了对象，但是没有使用）

      1. 构造器私有化（防止对象直接new）

      2. 类的内部创建对象（该对象是静态的）

      3. 向外暴露一个静态的公共方法，返回静态7对象（getInstance（））

        一般都是重量级对象

      4. 向外实现

    2. 懒汉式

      1. 仍然将构造器私有化
      2. 定义一个静态属性对象
      3. 提供一个公共的 public 的 static 方法 可以返回一个 对象，如果此时还没有创建对象，则创建一个对象
      4. 懒汉式只有当用户使用 这个方法的时候，才返回一个对象，后边再次调用时，会返回上次创建的对象，从而保证了单例

    3. 区别：

      1. 创建对象时机不同
      2. 饿汉式不存在线程安全问题，懒汉式存在线程安全问题（同时有三个程序调用方法，且同时触发判断，就会产生风险）
      3. 饿汉式存在浪费资源的问题
      4. javaSE标准类中，Java.lang.Runtime就是经典的单例模式

final关键字

基本介绍：

  1. 可以修饰类、属性、方法和局部变量
  2. 当不希望类被继承时，可以使用 final 修饰
  3. 当不希望父类的某个方法被子类覆盖 / 重写时，可以用 final关键字 修饰
  4. 当不希望类的某个属性的值被修改，可以用 final 修饰
  5. 当不希望某个局部变量被修改，可以使用 final

使用细节：

  1. final 修饰的属性又叫常量 XX_XX_XX来命名

  2. final 修饰的属性在定义时，==必须赋初值==，并且以后不能在修改，赋值可以在如下位置之一

    1. 定义时
    2. 在构造器中
    3. 在代码块中

  3. 如过 final 修饰的属性时静态的，则初始化的位置只能是

    1. 定义时
    2. 在静态代码块，不能在构造器中赋值

  4. final 修饰的类不能被继承，但是可以实例化对象

  5. 如果类不是 final 类，但是含有final方法，则方法虽然不能重写，但是可以被继承

  6. 如果一个类已经是 final 类了，那么类中的方法就没必要用 final 修饰了

  7. final 不能修饰构造方法

  8. final 和 static 往往搭配使用，效率更高，不会导致类的加载，底层编译器做了优化处理

  9. 包装类（Integer，Double，Float，Boolean等都是 final ）String 也是 final 类

抽象类

说明：

  1. 当父类的某些方法，需要声明，但是又不确定如何实现时，可以将其声明为抽象方法，那么这个类就是抽象类
  2. 当一个类中存在抽象方法（==不能有方法体==）时，需要将该类声明为 abstract 类
  3. 一般来说，抽象类会被继承，由他的子类完成抽象方法
  4. abstract 关键字修饰一个类或者方法时，这个类或者方法就叫做抽象类或抽象方法
  5. 抽象类时由设计师设计好后，让子类继承并实现抽象类()
  6. 在框架和设计模式使用较多

细节讨论：

  1. 抽象类 不能被实例化

  2. 抽象类 可以没有抽象方法还可以有实现的方法

  3. 一旦有了抽象方法，那么这个类必须声明为 abstract

  4. abstract 只能修饰类和方法，不能修饰属性等其他成员

  5. 抽象类 可以有任意成员==【抽象类的本质还是类】==

  6. 抽象方法不能有主体，即不能实现（不能加 {} ）

  7. 如果一个类继承了抽象类，则它==必须实现抽象类的所有抽象方法==，除非他自己也声明为抽象类

    所谓实现方法，就是有方法体，从语法上来说，只要加了 {} 就实现了该抽象方法

  8. 抽象方法不能使用 private、final、static 来修饰，因为这些标识符都和方法重写规则相违背

抽象类的最佳实践 – 模板设计模式

需求：

  1. 有多个类，完成不同的工作（job）
  2. 要求统计得到各自完成任务的时间
  3. 编程实现

实现：

  1. 设计一个抽象类 Template
  2. 编写方法CalculateTime() 方法，可以计算某段代码运行所耗费的时间
  3. 编写抽象方法 job()
  4. 编写一个子类 Sub，继承抽象类 Template ，并实现 job() 方法
  5. 编写一个 test 类，测试是否好用

接口（interface）

说明：接口就是给出一些没有实现的方法，封装到一起，到某个类要使用的时候，再根据具体情况把这些方法写出来

具体语法：

小结：

  1. 在 jdk7.0 以前 接口里的所有方法都没有方法体，==即都是抽象方法==

  2. jdk8.0 后接口类可以有==静态==方法，==默认==方法，也就是说接口中可以有方法的具体实现

    1. 默认方法

      ==需要用 default 关键字修饰，否则报错==

    2. 静态方法

      正常写就行

注意事项和相关细节：

  1. 接口不能被实例化

  2. 接口中的所有方法都是==public 方法== 接口中的抽象方法 可以不用 abstract 修饰

  3. 如果一个类 implements 实现接口，那么这个类需要将该接口的==所有抽象方法==都实现（可以使用 alt + enter 快捷创建方法）

  4. 使用抽象类实现接口，可以不用实现接口的方法

  5. 一个类可以实现多个接口

    class 类名 implements 接口1,接口2{ }

  6. 接口中的属性都是 final 的，并且是 public final static 修饰符（所有的属性都被这三个修饰符修饰）

  7. 接口中属性的访问形式（由第六点决定）

    接口名 . 属性名

  8. 一个接口不能继承其他的类（接口和类是实现的关系），但是可以继承多个别的接口

    interface A extends B,C{ }

  9. 接口的修饰符，只能是 ==public== 和 ==默认== ，这点和类的修饰符是一样的

实现接口和继承类：

  1. 当子类继承了父类，就会自动拥有父类的功能 —— 继承
  2. 如果子类需要扩展功能，可以通过实现接口的方式扩展
  3. 可以理解为==实现接口是对 java 单继承机制的一种补充==

区别：

  1. 接口和继承解决的问题是不一样的

    1. 继承的价值主要在于：解决代码的复用性和维护性

    2. 接口的价值主要在于：设计，设计好各种规范（方法），让其它类来实现这些方法

      即更加灵活

  2. 接口比继承更加灵活

    1. 继承是满足 is - a 的关系
    2. 而接口只需要满足 like - a 的关系

  3. 接口在一定程度上实现 代码解耦【即：接口的规范性 + 动态绑定机制】

接口的多态特性：

  1. 多态参数（接口引用可以指向实现了接口的类的对象）

  2. 多态数组

  3. 接口的多态传递

内部类（助看底层源码）

基本介绍：一个类的内部又完整的嵌套了另一个类结构，被嵌套的类称为内部类（inner class），嵌套其他类的类称为外部类（outer class ），是我们类的第五大成员【成员变量（属性）、成员方法、构造器、代码块、内部类】，内部类最大的特点就是==可以直接访问私有属性，并且可以体现类与类之间的包含关系==

基本语法：

分类：

  1. 定义在外部类局部位置上：

    1. 局部内部类（有类名 ）

      说明：局部内部类是定义在外部类的==局部位置==，比如方法中，并且有类名

      1. 可以直接访问外部类的所有成员，包含私有的

      2. 不能添加访问修饰符，因为他的地位就是一个==局部变量==，局部变量不能使用修饰符修饰，但是可以使用 final 修饰

      3. 作用域：仅仅在定义它的方法或者代码中

      4. 外部其他类，不能访问局部内部类

      5. 如果外部类和局部内部类成员重名时，默认==遵循就近原则==，如果想访问外部类的成员，则可以使用==（外部类 . this . 成员）==去访问

        eg : System.out.println("外部类的 n2= " + Outer.this.n2)

        Outer.this.n2)

        解释：Outer.this 本质上就是一个外部类的对象，即哪个对象调用了 m1 ，Outer.this 就指向哪个对象

        Ps：可以使用hashcode码来验证

    2. 匿名内部类（没有类名 重点！！！）（AnonymousInnerClass）

      说明：匿名内部类是定义在外部类的==局部位置==，比如方法中，并且没有类名，涉及到知识点有==（多态、动态绑定、内部类、继承）==

      本质：（1）是一个类；（2）是一个内部类；（3）该类匿名；（4）同时是一个对象

      1. 匿名内部类的基本语法：

      2. 基于接口、类、抽象类创建匿名内部类的方法以及相应的规则

        1. 匿名内部类既是一个类的定义，同时它本身也是一个对象，因此从语法上看，它既有定义类的特征，也有创建对象的特征，因此可以调用匿名内部类方法
        2. 可以直接访问外部类的所有成员，包含私有的
        3. 不能添加访问修饰符，因为它的对象就是一个局部变量
        4. 作用域：仅仅在定义它的方法或者代码块中
        5. 匿名内部类 —— 访问 ——> 外部类成员 [ 访问方式 ：==直接访问== ]
        6. 外部其他类 —— 不能访问 —–> 匿名内部类（因为匿名内部类地位是一个局部变量）
        7. 如果外部类和内部类的成员重名时，内部类访问的话，默认遵循就近原则，如果像访问外部类的成员，则可以使用（外部类名.this.成员）去访问。

      3. 匿名内部类的最佳实践

        当做==实参==直接传递，简洁高效

  2. 定义在外部类的成员位置上：

    1. 成员内部类（没有 static 修饰）

      说明：成员内部类是定义在外部类的==成员位置==，并且没有 static 修饰

      规则：

      1. 可以直接访问外部类的所有成员，包括私有的

      2. 可以添加任意访问修饰符（public、private、protected、默认），因为他的地位就是一个成员

      3. 作用域：为整个类体

      4. 成员内部类—— 访问 —–>外部成员 直接访问

      5. 外部类 ——– 访问 ——> 成员内部类 先创建对象，在访问

      6. 外部其他类 ——- 访问 ——> 成员内部类

        1. 第一种方式

        1. 在外部类中，编写一个方法，可以返回 Inner08 对象

      7. 外部类和成员内部类成员重名时，和前面的处理方式相同

    2. 静态内部类（使用 static 修饰）

      说明：静态内部类是定义在外部类的==成员位置==，并且==有 static 修饰==

      1. 可以直接访问外部类的所有静态成员，包括私有的，但不能直接访问非静态成员

      2. 可以添加任意访问修饰符，因为他的地位就是一个成员

      3. 作用域：同其他成员，为整个类体

      4. 访问方式

        1. 方式一

        2. 方式二（创建方法返回对象）

      5. 如果外部类和静态内部类的成员重名时，遵循就近原则，如想访问外部类成员，则可以使用（外部类名.成员）去访问
