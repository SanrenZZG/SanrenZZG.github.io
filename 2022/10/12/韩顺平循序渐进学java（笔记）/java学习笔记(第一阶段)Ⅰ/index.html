字符编码表

编码名称 具体功能
Unicode 固定大小的编码，使用两个字节来表示字符，字母和汉字都是用两个字节（兼容ASCII码）
ASCII 只占用一个字节的后七位，一共128字符，一个字节最多可以存储256个字符
utf-8 大小可变的编码，字母使用一个字节，汉字使用三个字节（Unicode码的一种改进）
GBK 可以表示汉字，而且范围很广，字母使用一个字节，汉字使用三个字节
GB2312 可以表示汉字 且gb2312 < gbk
big5 繁体中文， 香港、台湾

5.自动类型转换

​ （1）数据类型按精度（容量）大小排序为：

​ char=>int=>long=>float=>double

​ byte=>short=>int=>long=>float=>double

​ （2）自动类型转换细节

​ 有多种类型的数据混合运算时，系统首先启动所有数据转换成容量最大的那种数据类型，然后再进行运算。

​ 当我们把精度（容量）大的数据类型赋值给精度（容量）小的数据类型时，就会报错，反之会自动转换。

​ byte、short、char 三者都能计算，在计算时首先转换为 int 类型，三者相互之间不能自动转换

​ boolean不参与转换。

​ 自动提升原则：表达式的结果的类型自动提升为，操作数中最大的类型。

6.基本数据类型的强制类型转换

自动类型转换的逆过程，将容量大的数据类型转换为容量小的数据类型。使用时要使用强制转换符 （）

可能造成精度降低或溢出

短路与 && 与逻辑与 & 的区别：

​ 在==短路与==的判断过程中，如果第一个条件为==假==，则==不会继续检验第二个条件==，效率较高

​ 在==逻辑与==的判断过程中，如果第一个条件为==假==，则仍然判断第二个条件是否为真，效率较低

​ 在开发中多使用短路与，尽量不使用逻辑与

短路或 || 与逻辑或 | 的区别：

  • 在==短路或==的判断过程中，如果第一个条件为true，则==直接跳过第二个条件==，输出为true，效率较高
  • 在==逻辑或==的判断过程中，如果第一个条件为true，则==第二个条件也会执行==，效率较低
  • 开发过程中，大多会用短路或

八、标识符的命名的规则和规范

标识符概念：

java对各种变量、方法和类等命名时使用的字符序列称为标识符

标识符规则：

  • 由26个英文字母大小写，0~9，或 $ 组成
  • 数字不可以开头
  • 不可以使用关键字，但能包含关键字和保留字
  • java中严格区分大小写，长度没有限制
  • 标识符不能包含空格

标识符命名规范：

  • 包名：多单词组成时所有字母都小写：aaa.bbb.ccc
  • 类名、接口名：多单词组成时，所有单词的首字母大写：XxxYyyZzz【大驼峰】
  • 变量名、方法名：多单词组成时，第一个单词字母小写，第二个单词开始每个单词首字母大写：xxxYyyZzz【小驼峰】
  • 常量名：所有单词都大写，多单词时每个单词用下划线链接：XXX_YYY_ZZZ

位运算：

运算符 表达意义 具体方法
& 按位与 两位全为 1 ，结果为 1 ，否则为 0
| 按位或 两位有一个 1 ，结果为 1 ，否则为 0
^ 按位异或 两位一个为 0 一个为 1 ，结果为 1 ，否则为 0
~ 按位取反 0 => 1 , 1 => 0 （符号位也要取反）

~ - 2 =?

原码：10000000 00000000 00000000 00000010

反码：11111111 11111111 11111111 11111101

补码：11111111 11111111 11111111 11111110

按位取反：00000000 00000000 00000000 00000001

取反后的原码：00000000 00000000 00000000 00000001

得 1

原码、反码、补码：

  1. 正数的原码、反码、补码都一样；
  2. 负数的反码，除符号位其他各位取反；
  3. 负数的补码等于负数的反码+1；
  4. 0的反码和补码都是0
  5. 计算机运算时，都是以==补码==的形式计算的
  6. Java没有无符号数，换言之，Java中的数都是有符号的
  7. 当我们看运算结果时，要看他的原码！！

位运算符：

符号 具体作用
>>（算数右移） 低位溢出，符号位不变，并用符号位补溢出的高位（1 >> 2 相当于1 / 2 / 2）
<<（算数左移） 符号位不变，低位补 0 （1 << 2 相当于1 * 2 * 2）
>>>（逻辑右移 或 无符号右移） 低位溢出，高位补 0
