第七章 面向对象编程（基础）

类与对象（OOP）

说明：

  1. 类就是数据类型
  2. 对象就是一个类的一个具体的实例，是具体的实际的
  3. 类是抽象的，概念的，代表一类事务
  4. 类是对象的模板，对象是类的一个个体，对应一个实例

对象在内存中的存在形式：

属性/成员变量

  1. 从概念或叫法上看：==成员变量 = 属性 = filed==（即成员变量是用来表示属性的）
  2. 属性是类的一个组成部分，一般是基本数据类型，也可以是引用类型（对象、数组）

注意事项：

  1. 属性的定义语法和变量一样 示例：访问修饰符 属性类型 属性名

访问修饰符：控制属性的访问范围

四种访问修饰符：public、protected、默认、private

  1. 属性的定义类型可以为任意类型，包含基本类型和引用类型
  2. 属性如果不赋值，有默认值，规则和数组一致

创建对象的两种形式：

  1. 先声明后创建：Cat cat；cat = new Cat（）；
  2. 直接创建

如何访问属性：

​ 语法：对象名 . 属性名

内存分配机制

类与对象的内存分配机制：

​ 定义类时，会先在方法区内加载类信息（属性信息、方法信息），new 会在堆内开辟对应空间，然后将开辟空间的地址传到栈内给了对应的对象名，当对象内属性为字符串类型，将会存储在==方法区内的常量池==中，整型会存储在==堆中的存储空间==内。此时新创建一个对象名指向旧对象名，则此时新旧对象名都指向==同一个空间==

java内存结构分析：

  1. 栈：一般存放一本数据类型（局部变量）
  2. 堆：存放对象（Cat、cat、数组等）
  3. 方法区：常量池（常量、比如字符串）、类加载信息

流程分析：

  1. 先加载类信息（只加载一次）
  2. 在堆中分配空间，进行默认初始化（看规则），
  3. 把地址赋给 p ，p 就指向对象
  4. 进行指定初始化（给属性赋值） 比如 ：p.name = “jake”;

成员方法

介绍

在某些情况下，我们需要定义成员方法（简称方法）；

使用：先创建对象，然后直接调用方法即可

方法的调用机制：

  1. 当程序执行到方法时，就会开辟一个独立的空间（栈空间）
  2. 当一个栈（方法）执行完毕后，或者执行到 return 语句时，就会返回
  3. 返回到方法调用的地方，开辟的独立空间销毁
  4. 返回后，继续执行方法后面的代码
  5. 当 main 栈执行完毕后，整个程序退出

成员方法的优点：

  1. 提高代码的复用性
  2. 可以将实现的细节封装起来，然后供其他用户来调用即可

成员方法的定义：

  1. 形参列表：表示成员方法输入
  2. 返回数据类型：表示成员方法输出
  3. 方法主体：表示方法体，为了完成某一个功能的代码块
  4. return语句：语句不是必须的（如果有具体的返回数据类型，则必须有 return）
  5. 访问修饰符：控制方法使用的范围，有四种（如果不写，就是 默认 访问修饰符） public、protected、默认、private

使用细节

  1. 返回数据类型：

    1. 一个方法最多只能有一个返回值（如果要返回多个结果，可以返回数组）
    1. 返回类型可以为任意类型，包含基本类型和引用类型（如上）
    2. 如果方法要求有返回数据类型，则方法体中最后的执行语句必须为 return 值；而且要求返回值类型必须和 return 的值的类型一致
    3. 如果返回值类型为 return ，可以不写 return 语句，或者只写 return
    4. 方法名 遵循驼峰命名法，表达出该功能的意思即可

  2. 形参列表

    1. 一个方法可以有0个参数，也可以有多个参数，中间用逗号隔开
    2. 参数类型可以为任意类型，包含基本类型和引用类型
    3. 调用参数的方法时，一定要对应着参数列表传入相同类型或兼容类型的参数
    4. 方法定义时的参数称为形式参数，简称==形参==；方法调用时的参数称为实际参数，简称==实参==，实参和形参的类型要==一致或兼容==，个数和顺序必须一致

  3. 方法体

    1. 里面写完成功能的具体语句，可以为输入、输出、变量、运算、分支、循环、方法调用、但是不能在定义方法（即方法不能嵌套定义）

  4. 方法调用相关细节

    1. 同一个类的方法可以直接调用：直接调用即可

    2. 跨类中的方法 A 类调用 B 类方法：需要创建对象名

    3. 跨类的方法调用需要考虑访问控制符

成员方法的传参（parameter）机制 ！

  1. 基本数据类型的传参机制

    1. 基本数据类型，传递的是值（值拷贝），==形参的任何改变不影响实参==

  2. 引用数据类型的传参机制

    1. 引用类型（对象、数组）传递的是地址（传递也是值，但值是地址），==可以通过形参影响实参==

方法递归调用（recursion）！

基本说明：

递归重要规则：

  1. 执行一个方法时，就创建一个==新的受保护的独立空间（栈空间）==
  2. 方法的局部变量是==独立的==，不会互相影响，比如 n 变量
  3. 如果方法中使用的是==引用类型变量（比如数组）==，就会==共享引用类型的数据==
  4. 递归==必须向退出递归的条件逼近==，否则就是无限递归，出现（StackOverflowError，死龟了）
  5. 当一个方法执行完毕，或者遇到 return ，就会返回，==遵守谁调用，就将结果返回给谁==，同时当方法执行完毕或者返回时，该方法也就执行完毕

方法重载

介绍：java中允许同一个类中多个同名方法的存在，但要求 ==形参列表不一致！==

例子：System.out.println(); out 是 PrintSteam 类型

好处：

  1. 减轻了起名的麻烦
  2. 减轻了记名的麻烦
  3. 利于接口编程

注意事项：

  1. 方法名一定要一样

  2. 形参列表：必须不同（类型或者个数或者顺序不一样，至少有一样不同，参数名无要求 与方法重载定义无关）

  3. 返回值类型：无要求 与方法重载定义无关

可变参数

介绍：java允许将同一个类中==多个同名同功能但参数个数不同==的方法，封装成一个方法

就可以通过可变参数实现

基本语法：

注意事项：

  1. 可变参数的实参可以为 0 个或者任意多个
  2. 可变参数的实参可以为数组
  3. 可变参数的==本质就是数组==
  4. 可变参数可以和普通类型的参数一起放在形参列表，但必须保证==可变参数在最后==
  5. 一个形参列表中==只能出现一个==可变参数

作用域！

  1. 在java编程中，主要的变量就是局部变量和全局变量
  2. 局部变量一般是指==在成员方法中定义==的变量
  3. 全局变量：也就是属性，作用域为==整个类体==
  4. 局部变量：也就是除了属性之外的其他变量，作用域为==定义它的代码块中==
  5. 全局变量（属性）==可以不赋值==，直接使用，因为有默认值，局部变量==必须赋值==后，才能使用，因为没有默认值

使用细节：

  1. 属性和局部变量可以重名，访问时遵循==就近原则==

  2. 在同一个作用域中，比如在同一个成员方法中，两个局部变量，不能重名

  3. 属性生命周期较长，伴随着对象的创建而创建，伴随着对象的销毁而销毁，局部变量生命周期较短，伴随着他的代码块执行而创建，随着代码块的结束而销毁，即在一次方法的调用中生效。

  4. 作用域的范围不一样

    1. 全局变量/属性：可以被本类使用，也可以被其他类使用（通过对象调用）
    2. 局部变量：只能在本类中的对应方法中使用

  5. 修饰符不同

    1. 全局变量/属性 可以加修饰符（public protected pravate ….）
    2. 局部变量不可以加修饰符

构造器(constructor)

基本语法：

说明：

  1. 构造器的修饰符==可以默认==
  2. 构造器==没有返回值==
  3. ==方法名== 和 ==类名== 必须一样
  4. 参数列表 和 成员方法 一样的原则
  5. 构造器的调用是由==系统==完成的

基本介绍：构造方法又叫构造器，是类的一种特殊方法，它的主要作用是==完成对新对象的初始化==

  1. 方法名和类名相同
  2. 没有返回值
  3. ==在创建对象时，系统会自动的调用该类的构造器完成 对 对象的初始化==

this 关键字

介绍：java虚拟机会给每一个对象分配一个 this，this 代表当前对象，简单地说，==哪个对象调用，this 就指向哪个对象==

注意事项：

  1. this关键字可以用来访问本类的属性、方法、构造器

  2. this用于区分当前类的属性和局部变量

  3. 访问成员方法的语法：this.方法名（参数列表）

  4. 访问构造器语法：this（参数列表）；注意==只能在构造器中使用==

  5. this 不能在类定义的外部使用，==只能在类定义的方法中使用==
